TEXT DOCUMENT ANALYZER USER GUIDE

The Text Document Analyzer is a stand alone application for parsing literary documents and extracting statistical and mathematical metrics from the written work. Extracted data is stored in an integrated database so that it may be recalled at the user's convenience. Data records can be added to the database, deleted from the database, or displayed using the GUI's menus and controls.

1.1 DATABASE RECORD

Records in the database consist of metadata entered by the user and metrics generated when a document is parsed by the application. Each record correlates to a single literary document that has been parsed by the application. Records can be added and deleted by the user through the use of the application's menu system.

Metadata fields stored in a record include:

  • Author - the author of the document
  • Title - the title of the document
  • Year published - the year the document was published
  • Era - the era, BCE or ACE, the document was written
  • Genre - the genre of the document

Metrics fields stored in a record include:

  • Word Count - number of words
  • Punctuation Count - number of punctuation marks
  • Sentence Count - number of sentences
  • Syllable Count - number of syllables
  • Distinct Word Count - number of unique words
  • Average Word Length - average number of letters in a word
  • Average Words Per Sentence - average number of words in a sentence
  • Average Syllables Per Word - average number of syllables in a word
  • Distinct Words List - a list of unique words and their frequency
  • Distinct Punctuation List - a list of unique punctuation marks and their frequency
  • Flesch Reading Ease Score - a score relating the the reading level of the document
    • 90-100: Very easy to read
    • 80-90: Easy to read
    • 70-80: Fairly easy to read
    • 60-70: Plain English
    • 50-60: Fairly difficult to read
    • 30-50: Difficult to read
    • 10-30: Very difficult to read
    • 0-10: Extremely difficult to read

1.2 HOW TO: UPLOAD TEXT DOCUMENT FOR ANALYSIS

Uploading a text document for analysis can be done using the applications menu system and input components. Documents uploaded will be parsed by the application for metrics and all calculated data will be stored in the application database.

Note: Author, Title, Era, and Genre are required. Publish Year may be blank.

  1. Select File > Add Entry from the application's top menu. A new window will appear where you may browse for a file and enter metadata about the document.
  2. Enter the Title, Author, Publish Year, Era, and Genre.
  3. Click the Browse button.
  4. Locate and select the document you would like to parse.
  5. Click OK.
  6. Click the Parse button.
  7. A dialog box will appear to confirm the document has been parsed.

1.3 HOW TO: DELETE AN ENTRY FROM THE DATABASE

Deleting an entry from the database can be done using the application's top menu. Entries must be deleted one at a time. Entries deleted from the database cannot be recovered and must be recreated by reparsing the original text document.

  1. Select File > Delete Entry from the application's menu.
  2. Use the dropdown menu to select the document record you would like to delete. To delete all records at once, select the DELETE ALL RECORDS option.
  3. Select the Delete button.
  4. Click Yes to confirm that the record(s) should be deleted, or No to cancel.
  5. A dialog box will appear to confirm the document has been deleted or preserved.

1.4 HOW TO: VIEW ANALYZED DOCUMENT'S METRICS

After parsing a text document, data can be retrieved from the database and displayed in the application's display panel. All metadata and metrics stored in the document's database record will be displayed.

  1. Use the dropdown menu to select the document you would like to view.
  2. Click the Display button.
  3. The metrics will appear in the display panel at the bottom of the window.

1.5 HOW TO: QUERY THE DATABASE

The Text Document Analyzer can also be used to perform more advanced queries on the literary data stored in the database. Though this information is not stored in a single database record, the application can aggregate data from multiple records sharing a common field. For example, using the Query Database window a user can ask for the average word count of all records where the author is Edgar Allan Poe.

  1. Click File > Query Database in the application's top menu.
  2. Use the dropdown menus and the textbox to construct your query.
  3. Click the Display button.
  4. The results will appear in the display panel at the bottom of the window.

VIEW - Statistical and mathematical values such as Average, Total, Min, and Max can be selected using the VIEW dropdown menu.

OF - Metrics such as Word Count and Flesch Reading Ease Score can be selected using the OF dropdown.

WHERE - Criteria such as Author or Genre can be selected using the WHERE dropdown.

EQUALS - The EQUALS text field allows the user to enter a custom value corresponding to the WHERE dropdown option, such as the value "Edgar Allan Poe" for the Author option or "Comedy" for the Genre option.

1.6 HOW TO: SEARCH FOR DISTINCT WORDS

Once a document has been parsed, The Text Document Analyzer stores a list of every distinct word used in the document, as well as how many times each word was used. The Search Distinct Words window can be used to view this information. This functionality can be used to determine how many times the word "love" appears in Shakespeare's play "Much Ado About Nothing".

  1. Select File > Search Distinct Words from the application's top menu.
  2. Use the dropdown menu to select a record to search.
  3. Enter a word to search for into the textfield, or enter * to view a list of all distinct words.
  4. Click the Display button.
  5. The results will appear in the display panel at the bottom of the window.
